# Azure SQL

Azure SQL stores the governed metadata and structured credit-risk analytics tables.

## Demo Configuration

The dev database uses Azure SQL Database serverless:

```text
Service tier: General Purpose serverless
SKU: GP_S_Gen5_1
Min capacity: 0.5 vCore
Auto-pause delay: 60 minutes
Max size: 1 GB
Backup storage redundancy: Local
Region: Canada Central
```

This is suitable for intermittent demo usage. Microsoft documents that when a
serverless database is paused, compute cost is zero and only storage is billed.

Reference:

```text
https://learn.microsoft.com/en-us/azure/azure-sql/database/serverless-tier-overview
```

## Load Tables

Run the loader with SQL connection values supplied through environment variables:

```bash
SQL_SERVER="<server>.database.windows.net" \
SQL_DATABASE="sqldb-crai-dev" \
SQL_USER="<admin-login>" \
SQL_PASSWORD="<admin-password>" \
python3 scripts/load_sql_tables.py
```

Expected row counts:

```text
document_registry: 23
ifrs9_credit_risk: 29465
ifrs9_default_rate_by_grade: 7
ifrs9_default_rate_by_intent: 6
ifrs9_default_rate_by_home_ownership: 4
```

Verify loaded counts:

```bash
python3 scripts/query_sql_counts.py
```

## Security Notes

- The database is for synthetic/demo data only.
- The SQL admin password is generated by Terraform and is not committed.
- Local loading uses a narrow firewall rule for the current client IP.
- Production systems should use managed identity and least-privilege database roles.

## Region Note

The core demo resource group is in East US 2. Azure SQL is configured separately
because SQL provisioning can be restricted by subscription/region. Canada Central
is used for the SQL demo database when East US 2 is unavailable.
